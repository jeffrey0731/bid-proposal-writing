# 采购代理机构比选服务方案编写 Skill

## 核心原则

**评分标准决定结构，采购需求决定内容，合同条款决定边界。**

服务方案的首要任务是逐项、精准地响应比选文件的评审标准和采购需求（含合同条款），其次才是从专业角度补充差异化内容。任何脱离评审标准的"自说自话"都是无效内容。

---

## 工作流程

### 第零步：信息收集

向用户索取以下材料（缺少任何一项都会影响方案质量，应主动提醒）：

**必需材料：**
1. 比选文件全文（含附件、答疑澄清文件）
2. 采购需求或技术规格书
3. 合同条款或合同范本

**强烈建议提供：**
4. 项目背景资料（立项批复、可研报告等）
5. 采购人单位信息（性质、层级、行业）
6. 代理机构的公司简介、业绩、团队等素材

**如有则提供：**
7. 同类项目的历史方案
8. 采购人的特殊要求或偏好
9. 竞争对手信息

如果用户未提供比选文件，必须先索取，不可凭空编写。

### 第一步：比选文件深度分析

读取比选文件后，按以下四个维度提取信息，以结构化表格形式呈现给用户确认：

#### 1.1 评审标准拆解（最关键）

制作**评审标准响应对照表**，格式如下：

| 序号 | 评审项 | 分值 | 评审要点 | 响应策略 | 方案对应章节 |
| ---- | ------ | ---- | -------- | -------- | ------------ |

逐条拆解每一项评分标准，明确：
- 该项考的是什么（能力？方案？价格？业绩？）
- 得分的充分条件是什么
- 我们准备写什么内容来响应
- 内容放在方案的哪个章节

#### 1.2 采购需求分析

提取比选文件中的采购需求，区分：
- **硬性需求**（必须满足，否则废标）
- **软性需求**（满足程度影响得分）
- **隐性需求**（文件未明说但合情合理的期望）

#### 1.3 合同条款分析

重点关注合同中的：
- 付款条件与节点
- 违约责任与赔偿上限
- 服务期限与交付标准
- 验收标准与程序
- 知识产权归属
- 保密义务
- 不可抗力条款

合同条款决定了方案中承诺的边界——不要承诺合同中没有约定的责任，也不要遗漏合同中已经约定的义务。

#### 1.4 项目属性判定

确认以下信息，决定方案的专业基调：
- 采购方式（公开比选/邀请比选/其他）
- 采购属性（工程/货物/服务）
- 采购人性质（国家机关/事业单位/国有企业/其他）
- 适用法律体系（政府采购法体系/招标投标法体系/企业内部制度）
- 是否为依法必须招标项目

**将以上分析结果整理后提交用户确认，确认后方可进入第二步。**

### 第二步：框架结构设计

基于第一步的分析结果，设计方案框架。核心逻辑是：**评审标准中的每一项评分要素，都必须在框架中有明确对应的章节。**

#### 2.1 结构设计原则

1. **评审导向原则**：章节标题和顺序尽量与评审标准的评分项对应，让评委一眼就能找到打分点
2. **权重匹配原则**：评审标准中分值高的项，方案中篇幅要大；分值低的项，不要喧宾夺主
3. **合同覆盖原则**：合同中的关键条款（如付款、违约、验收）在方案中要有回应
4. **专业补充原则**：在满足评审要求的基础上，可增加体现专业深度的章节

#### 2.2 通用框架参考（需根据具体项目裁剪）

```
前言（简述对项目的理解和代理服务承诺）

第一章  项目理解与需求分析
  1.1 项目概况
  1.2 采购需求理解
  1.3 项目重难点分析
  1.4 代理服务工作思路
  （本章回应评审标准中"对项目的理解"类评分项）

第二章  代理服务方案
  2.1 服务内容与范围
  2.2 代理服务工作流程
  2.3 采购文件编制方案
  2.4 开评标组织方案
  2.5 合同签订及后续服务
  （本章是核心，回应评审标准中"服务方案"类评分项）

第三章  项目管理方案
  3.1 进度管理
  3.2 质量管理
  3.3 沟通协调
  3.4 档案管理
  （本章回应评审标准中"项目管理"类评分项）

第四章  人员配置方案
  4.1 项目组织架构
  4.2 岗位职责及人员配置
  4.3 人员培训与考核
  （本章回应评审标准中"人员配置"类评分项）

第五章  质量保障与风险控制
  5.1 质量管理体系
  5.2 关键节点质量控制
  5.3 风险识别与应对
  5.4 应急预案
  （本章回应评审标准中"质量保障"类评分项，也可合并到第三章）

第六章  增值服务与承诺
  6.1 增值服务内容
  6.2 服务承诺
  6.3 廉洁自律承诺
  （本章为专业补充内容，体现差异化竞争优势）

附件
  附件一：公司资质及业绩
  附件二：拟投入人员简历
  附件三：相关证明材料
```

#### 2.3 插图与表格计划

在框架中同步规划：
- 流程图（代理工作流程、开评标流程、文件编制流程等）
- 组织架构图（公司架构、项目团队架构）
- 进度甘特图
- 关键表格（业绩表、人员表、评分响应对照表等）

#### 2.4 篇幅规划

根据评审标准各项分值权重，分配各章节的目标字数。参考范围：
- 一般项目：60-120页，2万-6万字
- 重大项目：120-200页，6万-12万字

**框架设计完成后提交用户确认，确认后方可进入第三步。**

### 第三步：分阶段编写

编写分三个里程碑推进，每个里程碑内按章节逐步提交用户确认。

#### 里程碑一：框架确认（第二步的产出）

已在此步骤前完成。

#### 里程碑二：初步编写

按以下批次顺序编写：

**批次一（方向确认）：**
- 第一章：项目理解与需求分析
- 人员配置方案（如有评分要求）

先写这两章是为了确认对项目的理解方向是否正确，避免后续返工。

**批次二（核心内容）：**
- 第二章：代理服务方案
- 质量保障与风险控制

这是分值最重的核心章节，需要重点打磨。

**批次三（补充完善）：**
- 第三章：项目管理方案
- 第六章：增值服务与承诺
- 附件材料

每批次完成后提交用户确认，用户确认后方可继续下一批次。如用户未明确确认，不得自行推进，除非收到明确指令。

#### 里程碑三：扩写完善

根据用户对初稿的反馈：
1. 深化薄弱章节的内容
2. 补充案例、数据、图表
3. 优化文字表达
4. 统一格式排版
5. 进行评审标准响应的逐项复核——确保每一项评分标准都有对应的方案内容

**终稿完成后提交用户最终确认。**

## 编写规范

### 文字格式

- 正文：宋体/仿宋，小四号，1.5倍行距
- 一级标题：黑体三号，居中（第一章、第二章……）
- 二级标题：黑体四号，左对齐（1.、2.、3.……）
- 三级标题：黑体小四，左对齐（1.1、1.2……）
- 四级标题：宋体加粗小四（（1）、（2）……）
- 纸张：A4，页边距上下2.5cm，左3cm，右2.5cm

### 编号体系

严格按照中文公文格式：
- 第一级：第一章、第二章……
- 第二级：1.、2.、3.……
- 第三级：1.1、1.2、1.3……
- 第四级：（1）、（2）、（3）……
- 第五级：①、②、③……

### 图表规范

- 图编号：图X-X（章号-序号），标题在图下方居中
- 表编号：表X-X（章号-序号），标题在表上方居中
- 清晰度不低于300dpi
- 配色风格统一

---

## 编写时的注意事项

1. **不要自说自话**：每段内容都要能对应到评审标准的某一项。如果一段内容找不到对应的评分项，考虑删除或合并。
2. **不要面面俱到**：评审标准给10分的项写3页，给2分的项写半页。篇幅与分值匹配。
3. **不要空洞承诺**：少写"我们将提供优质服务"，多写"我们将在X个工作日内完成Y，具体措施为Z"。
4. **不要忽视合同**：合同中的付款节点、验收标准、违约责任，方案中必须有对应的服务安排。
5. **要突出针对性**：结合项目的行业特点、采购人属性、地域特征来写，避免"放之四海而皆准"的模板化内容。
6. **要体现专业性**：在响应评审标准的基础上，适当加入行业洞察、专业建议、增值服务，体现代理机构的专业深度。
7. **要控制节奏**：每个阶段、每个批次都等用户确认后再继续，避免方向偏差导致大面积返工。
